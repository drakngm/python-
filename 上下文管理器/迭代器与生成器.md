**迭代器**

可迭代的对象一定不能是自身的迭代器。也就是说，可迭代的对象
必须实现 __iter__ 方法，但不能实现 __next__ 方法。
另一方面，迭代器应该一直可以迭代。迭代器的 __iter__ 方法应
该返回自身。

**生成器**

生成器用于生成供迭代的数据
协程是数据的消费者
为了避免脑袋炸裂，不能把这两个概念混为一谈
协程与迭代无关
注意，虽然在协程中会使用 yield 产出值，但这与迭代无关

**装饰器负责函数级横切逻辑，上下文管理器负责代码块级资源管理。**


**一个简单地上下文管理器**
import logging
import time
from contextlib import contextmanager
from typing import Iterator


logger = logging.getLogger(__name__)


@contextmanager
def timer(task_name: str = "Task") -> Iterator[None]:
    """
    统计代码块执行时间。

    Args:
        task_name: 当前任务名称，用于日志输出。

    Example:
        with timer("Import Data"):
            import_data()
    """
    start_time = time.perf_counter()

    try:
        # yield 之前：进入 with 代码块
        yield
    finally:
        # yield 之后：退出 with 代码块（无论是否发生异常都会执行）
        elapsed = time.perf_counter() - start_time

        logger.info(
            "[%s] completed in %.4f seconds",
            task_name,
            elapsed,
        )

*简单运行实例*
import time

with timer("Generate Report"):
    time.sleep(2)
    print("正在生成报表...")

*输出*
正在生成报表...
INFO: [Generate Report] completed in 2.0012 seconds

**工作流程**
with timer():

↓

进入 timer()

↓

执行 yield 前面的代码
    ↓
    相当于 __enter__()

↓

执行 with 内部代码

↓

回到 yield 后面的代码
    ↓
    相当于 __exit__()

**开发时候应该如何选择？**

装饰器：当某个逻辑要应用到多个函数时，例如权限校验、重试、日志、函数计时。
上下文管理器：当某段代码需要自动管理资源时，例如数据库事务、文件、锁、临时修改配置、代码块计时等。

一个常见的企业代码组合就是：

@retry(max_attempts=3)
@permission_required("order:create")
def create_order(user):
    with timer("Create Order"):
        # 数据库事务
        # 调用第三方接口
        # 写日志
        ...
